# About the actual AZ-104 exam
* The exam questions are **very** different from the Practice exams provided on the Microsoft Learn website.
* The questions are long-winded, full of unnecessary details - with primary objective of confusing the candidate and throw the candidate off - not really testing the concepts. This is not how other reputable vendors (Oracle, Cisco, RedHat) test candidates on their exams, but this is how it is with Micorosft exams. The experience is going to be horrible - so brace yourself for it. Prepare well, with as much hands-on practice as possible.
* Just watching training videos will not be enough for you - no matter how good the quality of material in the videos.
* The exam / testing software from Pearson VUE will give you some tools that you are allowed to use during the exam. These including:
  * (Link to) **Microsoft Learn website** (Yes! You are allowed to open the link from within the exam software. There is a link to this website within the software - marked clearly). Don't use browser from your computer. 
  * Fun fact: (Microsoft copied this concept from RedHat and CNCF's exam offerings, e.g. the Certified Kubernetes Administrator exam.)

  * A **Calculator** 
  * A **whiteboard** - available only during the first scenario based question(s).
* Make sure that you develop good proficiency in using the Microsoft Learn website, so you can look up topics in the Microsoft Documentation without wasting time.
* On exam day, PearsonVUE will test your system for compliance. Make sure you login at least 30 minutes before the actual exam, because this testing can take very long time. You also have to have a margin of time in case something fails and you need to fix that before the exam starts.
* The only allowed national IDs are Driving License or Passport. Make sure you have them with you.

## Fun facts about tech vendors letting you use official documentation:

### The Pioneer: Red Hat (1999)
Red Hat is widely credited as the first major vendor to implement a "performance-based" (hands-on) exam format with the launch of the Red Hat Certified Engineer (RHCE) in 1999.

* From its inception, Red Hat’s philosophy was that in the real world, a sysadmin has access to manual pages (man pages) and built-in documentation.
* Candidates were (and still are) allowed to use the local documentation installed on the exam machine (e.g., `/usr/share/doc`) and the `man` pages. While they didn't provide a web browser to the public internet, all "official" product documentation was available locally.

### Cloud-Native - CNCF (2017)
The Cloud Native Computing Foundation (CNCF), through the CKA (Certified Kubernetes Administrator) launched in 2017, introduced the "Open Tab" model.

* Unlike Red Hat's "local-only" docs, the CNCF allowed candidates to open one additional browser tab to access the **live/online** official documentation at `https://kubernetes.io/docs`
* This set a new standard for modern DevOps exams, shifting the focus from memorizing YAML syntax to knowing how to find and apply the correct architecture patterns.

### Microsoft (2023)
For decades, Microsoft was known for strictly "closed-book" multiple-choice exams. However, in August 2023, Microsoft embraced the change.

* Microsoft now allows access to Microsoft Learn during all "role-based" certification exams (e.g., AZ-104, AZ-305).
* Unlike the CKA, you cannot use a search engine like Google; you are restricted to a specific interface that browses the `https://learn.microsoft.com` website.
