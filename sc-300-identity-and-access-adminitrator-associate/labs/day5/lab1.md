# Day 5 - Lab 1: Mixed-Domain Timed Simulation

## Context (Why this lab matters)
- This lab simulates exam-style context switching across SC-300 exam topics.

## Objective
Complete a timed identity operations run and capture evidence for each domain.

## Scenario
You are on-call and must handle multiple identity and governance tasks in one window.

## Direct Portal GUI Steps (Primary)
1. Perform one identity lifecycle task (user/group update).
2. Validate one authentication/access policy (CA or auth method scope).
3. Validate one workload identity object (app or service principal).
4. Validate one governance artifact (access review/package/workflow).
5. Record evidence for all four checks.

## PowerShell Path (Method #2)
- Re-run one verification command from Day 1-Day 4 per objective domain.

## Azure CLI Path (Method #3)
- Re-run one az ad or az rest command per objective domain.

## IaC Templates (Optional - ARM)
- ARM reference: labs/arm/day5/scenario01/main.json

## Prerequisites
- Baseline objects from previous labs are available.

## Sequence Notes
- Use a 60-minute timer.
- Do not make broad tenant policy changes during simulation.

## Validation
- Four domain checks completed.
- Evidence captured (command output or screenshots).

## Complete Cleanup
- Remove temporary simulation-only objects.
