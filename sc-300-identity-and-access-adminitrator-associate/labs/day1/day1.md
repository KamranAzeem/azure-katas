# Day 1 Overview

## SC-300 Objective Alignment
- Implement identities in Microsoft Entra ID: users, groups, administrative units, and lifecycle operations.
- Implement and manage external identities: B2B guest onboarding and access checks.
- Plan and implement entitlement basics: group-based access assignment patterns.

**Note:** Before Day 1, complete prerequisites in the main `README.md`.

## Lab Count
- Total labs: 4

## Labs and Estimated Time
1. [Lab 1](lab1.md) - User and Group Lifecycle Baseline
- Accomplishes: creates users/groups, validates membership, and confirms role assignment readiness.
- Estimated time (new to Entra): 45-65 minutes.

2. [Lab 2](lab2.md) - Administrative Units and Scoped Administration
- Accomplishes: scopes delegated administration with administrative units and limited roles.
- Estimated time (new to Entra): 40-60 minutes.

3. [Lab 3](lab3.md) - External Identity Guest Access Baseline
- Accomplishes: invites B2B guest users and validates collaboration controls.
- Estimated time (new to Entra): 35-55 minutes.

4. [Lab 4](lab4.md) - Group-Based Access Baseline for Azure Resources
- Accomplishes: maps Entra groups to Azure RBAC roles and validates effective access.
- Estimated time (new to Entra/Azure RBAC): 45-70 minutes.
